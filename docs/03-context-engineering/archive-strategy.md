# 文档归档策略（Archive Strategy）

## 目标

控制文档规模增长，避免上下文检索膨胀，同时保持历史可追溯。

## 总体原则

1. 主路径保留“当前有效知识”，历史内容迁入 `archive/`。
2. 任何归档都必须保留索引入口，不做“静默搬迁”。
3. AI 检索优先读索引和 overview，再按需打开归档条目。

## 触发条件（建议阈值）

- 单文档超过 400 行且持续增长。
- timeline 超过 120 条事件。
- TODO 列表跨 3 个迭代仍保留大量历史项。
- 某主题存在 3 次以上重大版本变更，且早期内容仅用于追溯。

## 归档目录建议

```text
archive/
  timeline/
  todo/
  decisions/
```

## 归档流程

1. 在原文档保留“当前摘要 + 历史入口链接”。
2. 将历史内容迁移到 `archive/` 对应子目录。
3. 更新 `memory/index.md` 增加归档入口关键词。
4. 在 `memory/timeline.md` 记录一次归档事件（可选但推荐）。

## 检索策略约束

- 默认不加载 `archive/` 全量文件。
- 仅当当前文档索引指向历史问题时，按链接定向读取归档文件。

## 反模式

- 将核心当前规范直接归档，导致主路径失真。
- 归档后不更新索引，导致“文件在但找不到”。
