# 安全、隐私与合规（Security & Privacy）

## 1. 数据分级（示例，按项目调整）

| 级别 | 描述 | 存储 | 日志 | 给模型 |
|------|------|------|------|--------|
| L0 公开 | 可公开文档 | 任意 | 可 | 可 |
| L1 内部 | 内部非敏感 | 受控仓库 | 摘要 | 可 |
| L2 敏感 | 业务敏感 | 加密 + ACL | 脱敏 | 需脱敏/最小化 |
| L3 高敏 | PII、密钥、健康/金融等 | 严格隔离 | **禁止**原文 | **默认禁止**或人工审批 |

## 2. 密钥与凭证

- **禁止**提交到 Git：使用密钥管理或环境注入
- Agent **默认无**生产写权限；需审批与 break-glass 流程

## 3. 供应链与依赖

- 固定版本或锁文件；已知漏洞需评估修复时限
- 对模型供应商做数据处理协议（DPA）审查（若适用）

## 4. AI 特有风险

- **提示注入（Prompt Injection）**：对用户/外源内容做边界标注与工具沙箱
- **数据渗出**：限制模型外发工具与白名单域名
- **越权工具调用**：最小权限 + HITL

## 5. 合规

- 列出适用法规：`GDPR`、`个人信息保护法`、行业规范等
- 数据保留期与删除流程（Right to erasure）

## 6. 事件响应

- 疑似泄露：**立即**按事故流程升级，保存证据链
- 使用 [templates/incident-template.md](../../templates/incident-template.md)

## 7. Reviewer 检查清单（摘要）

- [ ] 变更是否引入新的数据分级或跨境传输？
- [ ] 日志是否可能写入敏感字段？
- [ ] 新工具是否扩大网络出向？
